# Neovim エディタ要件定義書

この文書では、理想的なエディタに必要な機能を整理します。
各機能について「欲しい/不要」を決めてから、実際の設定に落とし込みます。

---

## 1. 基本的なエディタ機能

### 1.1 表示・見た目
- [x] 行番号を表示したい
- [x] カーソル行をハイライトしたい
- [ ] タブ・スペースを可視化したい
- [ ] カラースキーム（テーマ）を変更したい
- [ ] ステータスバー（下部の情報表示）が欲しい
- [ ] 全体的な見た目を自分好みにしたい

**メモ:**
```
どんな見た目が好き？
- 暗い背景 / 明るい背景
- シンプル / カラフル
- 最小限の情報 / 詳細な情報表示
```

### 1.2 文字入力・編集
- [ ] システムのクリップボードと連携したい（コピペ）
- [ ] 自動インデント（字下げ）が欲しい
- [ ] タブキーでスペース入力したい
- [ ] インデント幅を設定したい（2スペース？4スペース？）
- [ ] 行の自動折り返しは必要？

**メモ:**
```
普段どんな言語を書く？
- Ruby / Go / JavaScript / TypeScript / その他
→ 言語によって最適なインデント幅が異なる
```

## 2. ファイル管理機能

### 2.1 ファイルの探し方
- [ ] ファイルをツリー表示で見たい（左側に一覧）
- [ ] ファイル名で検索したい（曖昧検索）
- [ ] ファイルの中身を検索したい（grep検索）
- [ ] 最近開いたファイルから選びたい

**メモ:**
```
どんな使い方をする？
- 大きなプロジェクト / 小さなファイル編集
- よくファイル間を移動する / 1つのファイルに集中
```

### 2.2 複数ファイルの扱い
- [ ] 複数ファイルをタブで管理したい
- [ ] 画面分割して複数ファイルを同時に見たい
- [ ] 分割した画面間を素早く移動したい

## 3. コーディング支援機能

### 3.1 コード補完
- [ ] 入力中に候補を表示したい（自動補完）

- [ ] 言語固有の補完が欲しい（各言語のAPI等）

**重要度: 高 / 中 / 低**

### 3.2 構文チェック・エラー表示
- [ ] 書いている途中でエラーを教えて欲しい
- [ ] エラー箇所にジャンプしたい
- [ ] エラーの詳細を見たい

**重要度: 高 / 中 / 低**

### 3.3 定義ジャンプ
- [ ] 関数の定義元にジャンプしたい
- [ ] 変数が使われている場所を一覧表示したい
- [ ] ジャンプ後に元の場所に戻りたい

**重要度: 高 / 中 / 低**

### 3.4 コードフォーマット
- [ ] コードを自動整形したい
- [ ] 保存時に自動でフォーマットしたい
- [ ] 言語ごとのフォーマッタを使いたい

**重要度: 高 / 中 / 低**

### 3.5 シンタックスハイライト
- [ ] コードに色をつけて読みやすくしたい
- [ ] より高度な色分けが欲しい（関数/変数/型など）

**重要度: 高 / 中 / 低**

**重要:** 全部入りを目指さない！

- 動くようになったら1つずつ追加
- 理解しながら進める
